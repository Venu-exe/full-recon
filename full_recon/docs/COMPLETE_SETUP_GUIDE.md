# Complete Recon Toolkit Setup Guide

## What You Have

Your toolkit consists of **3 main scripts**:

| Script | Purpose | Runtime | Use Case |
|--------|---------|---------|----------|
| **daily_recon.sh** | Fast reconnaissance | 10-12 min | Daily bug bounty hunting (15-20 domains/day) |
| **full_recon.sh** | Comprehensive reconnaissance | ~60 min | Deep dives, thorough audits |
| **triage_enhanced.sh** | Auto-prioritize findings | 1-2 min | Generates priority list after recon |

## Step 1: Initial Setup (One-Time)

### 1.1 Create Working Directory
```bash
mkdir -p ~/bug-bounty/recon-toolkit
cd ~/bug-bounty/recon-toolkit
```

### 1.2 Copy All Scripts
```bash
# Copy main scripts
cp /path/to/daily_recon.sh .
cp /path/to/full_recon.sh .
cp /path/to/triage_enhanced.sh .
cp /path/to/triage.conf .

# Copy docs
mkdir -p docs
cp /path/to/README.md .
cp /path/to/TRIAGE_README.md docs/
cp /path/to/COMPLETE_SETUP_GUIDE.md docs/

# Make executable
chmod +x daily_recon.sh full_recon.sh triage_enhanced.sh
```

### 1.3 Install Go Tools

Required tools (all via Go):

```bash
# Subdomain enumeration
go install -v github.com/projectdiscovery/subfinder/v2/cmd/subfinder@latest
go install github.com/owasp-amass/amass/v4/cmd/amass@latest
go install github.com/d3mondev/puredns/v2@latest
go install github.com/Josue87/gotator@latest

# Live host discovery
go install -v github.com/projectdiscovery/httpx/cmd/httpx@latest

# Port scanning
go install -v github.com/projectdiscovery/naabu/v2/cmd/naabu@latest

# Crawling & URL discovery
go install -v github.com/projectdiscovery/katana/cmd/katana@latest
go install github.com/lc/gau/v2/cmd/gau@latest
go install github.com/tomnomnom/waybackurls@latest

# URL filtering (gf patterns)
go install github.com/tomnomnom/gf@latest

# JS/secret mining
go install github.com/lc/subjs@latest

# Screenshots
go install github.com/sensepost/gowitness@latest

# Vulnerability scanning
go install -v github.com/projectdiscovery/nuclei/v3/cmd/nuclei@latest
```

### 1.4 System Dependencies
```bash
# Ubuntu/Debian
sudo apt-get install -y jq curl git

# Arch Linux
sudo pacman -S jq curl git

# macOS
brew install jq curl git
```

### 1.5 Setup gf Patterns
```bash
# Create gf directory
mkdir -p ~/.gf

# Clone pattern repo
git clone https://github.com/1ndianl33t/Gf-Patterns ~/Gf-Patterns

# Copy patterns to gf directory
cp ~/Gf-Patterns/*.json ~/.gf/
```

### 1.6 Create Wordlists Directory
```bash
# For subdomain brute-forcing
mkdir -p ~/wordlists

# Download SecLists wordlist (or use your own)
cd ~/wordlists
wget https://raw.githubusercontent.com/danielmiessler/SecLists/master/Discovery/DNS/subdomains-top1million-110000.txt -O subdomains.txt
```

### 1.7 Create Scope Files (Optional but Recommended)
```bash
# For each target, create a scope file
mkdir -p ~/scopes

# Example: agoda.com in-scope subdomains
cat > ~/scopes/agoda.com_scope.txt << 'EOF'
agoda.com
.agoda.com
.agodahouse.com
EOF
```

## Step 3: Daily Usage

### Quick Recon (10-12 minutes)
```bash
cd ~/bug-bounty/recon-toolkit

# Run on a single domain
./daily_recon.sh agoda.com

# Recon runs automatically, then triage runs
# Results in: recon_agoda.com/triage/priority_list.txt
```

### View Results
```bash
# Text format (human-readable)
cat recon_agoda.com/triage/priority_list.txt

# JSON format (for tools)
cat recon_agoda.com/triage/priority_list.json | jq .

# Start testing (in priority order)
head -30 recon_agoda.com/gf_results/redirect.txt  # Test these first
head -30 recon_agoda.com/gf_results/xss.txt       # Then these
head -30 recon_agoda.com/gf_results/idor.txt      # Needs real session
```

### Parallel Testing (Multiple Domains)
```bash
# Test 3 domains in parallel (~40 min total instead of 120 min)
./daily_recon.sh agoda.com &
./daily_recon.sh booking.com &
./daily_recon.sh expedia.com &

wait

# Then check all results
for domain in agoda booking expedia; do
  echo "=== $domain.com ===" 
  cat recon_${domain}.com/triage/priority_list.txt
done
```

### Full Deep Dive (60 minutes)
```bash
# When you want comprehensive testing on a single target
./full_recon.sh agoda.com

# Generates all formats automatically:
# - Text, JSON, CSV, Markdown all created
```

## Step 5: Manual Testing Workflow

After running recon and triage:

### 1. Start with Open Redirects (Quickest)
```bash
cat recon_DOMAIN/gf_results/redirect.txt | head -20

# Test each:
# 1. Burp > open URL in browser
# 2. Intercept + change param to attacker.com
# 3. Check if redirect happens
# Average: 10 sec per URL, high success rate
```

### 2. Test XSS (Medium Difficulty)
```bash
cat recon_DOMAIN/gf_results/xss.txt | head -20

# For each URL with parameter:
# 1. Try: param=<img src=x onerror=alert(1)>
# 2. View page source for reflection
# 3. Try encoding if filtered
# Average: 1-2 min per URL
```

### 3. Test IDOR (Highest Value)
```bash
cat recon_DOMAIN/gf_results/idor.txt | head -20

# For each:
# 1. Login with your real account
# 2. Intercept request with ID in param
# 3. Change ID to different user's ID
# 4. Check if you can access their data
# Average: 2-5 min per URL (needs real account)
```

### 4. Test SSRF (Advanced)
```bash
cat recon_DOMAIN/gf_results/ssrf.txt | head -20

# For each:
# 1. Try: param=http://localhost:8080/
# 2. Try: param=http://127.0.0.1:22
# 3. Look for timing differences
# 4. Use Burp Collaborator for blind SSRF
```

## Step 7: Advanced Tips

### Tip 1: Skip Already-Tested Domains
```bash
# If you've tested agoda.com before, skip it
ls recon_*/ | grep agoda  # Check what you have
# Only run daily_recon on NEW targets
```

### Tip 2: Use NEW URLs First
```bash
# These are fresher, less-tested
cat recon_DOMAIN/urls/NEW_urls.txt | head -50

# Test these BEFORE looking at old URLs
```

### Tip 3: Combine Multiple Domains in CSV
```bash
# Run triage on multiple domains
./daily_recon.sh agoda.com
./daily_recon.sh booking.com
./daily_recon.sh expedia.com

# Combine results
(
  echo "Domain,Type,Parameter,URL,Severity"
  cat recon_agoda.com/triage/priority_list.csv | tail -n +2 | sed 's/^/agoda.com,/'
  cat recon_booking.com/triage/priority_list.csv | tail -n +2 | sed 's/^/booking.com,/'
  cat recon_expedia.com/triage/priority_list.csv | tail -n +2 | sed 's/^/expedia.com,/'
) > all_targets.csv

# Sort by severity, prioritize testing
sort -t, -k5 all_targets.csv
```

### Tip 4: Automate Daily Runs
```bash
# Create daily_targets.txt
cat > daily_targets.txt << 'EOF'
agoda.com
booking.com
expedia.com
EOF

# Run all at once
cat daily_targets.txt | while read domain; do
  ./daily_recon.sh "$domain" &
done
wait

echo "All domains complete!"
```

### Tip 5: Monitor for New Subdomains
```bash
# Each run creates NEW_subdomains.txt
# Test these FIRST (fresher = less-tested)

cat recon_DOMAIN/subdomains/NEW_subdomains.txt | while read sub; do
  echo "Testing new subdomain: $sub"
done
```

## Troubleshooting

### Problem: "command not found: subfinder"
```bash
# Make sure Go bin is in PATH
export PATH="$PATH:$(go env GOPATH)/bin"

# Add to ~/.bashrc to persist
echo 'export PATH="$PATH:$(go env GOPATH)/bin"' >> ~/.bashrc
source ~/.bashrc

# Re-run installation
go install -v github.com/projectdiscovery/subfinder/v2/cmd/subfinder@latest
```

### Problem: "httpx not found" - can't check live hosts
```bash
# Install httpx first
go install -v github.com/projectdiscovery/httpx/cmd/httpx@latest

# Re-run recon
./daily_recon.sh domain.com
```

### Problem: Triage script not found
```bash
# Make sure triage_enhanced.sh is in the same directory
ls -la triage_enhanced.sh
chmod +x triage_enhanced.sh

# Make sure both scripts are executable
chmod +x daily_recon.sh full_recon.sh triage_enhanced.sh
```

### Problem: "Can't find gf_results" in triage
```bash
# Make sure daily_recon.sh completed fully
# Check output directory exists:
ls -la recon_DOMAIN/gf_results/

# If empty, daily_recon.sh may have failed
# Check live_urls.txt was created:
wc -l recon_DOMAIN/http/live_urls.txt
```

### Problem: CSV export shows empty results
```bash
# Make sure you have enough URLs/findings
wc -l recon_DOMAIN/gf_results/*.txt

# If gf_results files are empty, no findings match patterns
# This is normal - just means no vulnerabilities found in gf patterns
```
